MyCritique

Site de critiques avec API REST (Symfony) et front-end (Vue.js)

Durée
~2 mois
Équipe
4 personnes
Architecture
Client-Serveur
⭐

Contexte du projet

Suite logique du projet MyAvatar, ce projet de SAE consistait à développer un site de critiques en architecture client-serveur. L'objectif était de créer deux applications distinctes qui communiquent entre elles : une API REST développée avec Symfony et API Platform, et une application front-end en Vue.js.

Le thème des objets critiquables était semi-libre — notre équipe a choisi de créer un site de critiques permettant aux utilisateurs de noter et commenter différents types d'objets.

Architecture du projet

  • Back-end (API REST) : Développé avec Symfony et API Platform, gère toute la logique métier, l'authentification JWT, et expose les ressources via des endpoints REST.
  • Front-end (Vue.js) : Application SPA qui consomme l'API, gère l'interface utilisateur et la navigation avec Vue Router.
  • Intégration MyAvatar : Les photos de profil sont chargées depuis le service MyAvatar en utilisant le hash SHA256 de l'email de l'utilisateur.

Fonctionnalités principales

  • Authentification JWT : Système complet de connexion/inscription avec tokens JWT et rafraîchissement automatique.
  • Gestion des rôles : Utilisateurs basiques et administrateurs avec permissions différenciées.
  • CRUD objets critiquables : Création, lecture, modification et suppression d'objets (validation requise pour les utilisateurs basiques).
  • Système de critiques : Notes, commentaires et critères spécifiques par objet avec calcul de moyenne.
  • Favoris : Mise en favori d'objets avec page dédiée listant les dernières critiques.
  • Administration : Gestion des comptes, validation des soumissions, suppression de contenu.

Ma contribution

Au sein de l'équipe, je me suis concentré sur :

  • Système de rôles et permissions : Implémentation des voters Symfony et configuration des accès API Platform avec le paramètre security.
  • Gestion des objets critiquables : Développement des endpoints pour la création et la gestion des objets.
  • Intégration front-end : Participation au développement des composants Vue.js liés aux fonctionnalités que j'ai développées côté API.
