Projet Trains

Développement complet du jeu de plateau japonais « Trains » en Java

Durée
1 semestre
Équipe
2 personnes
Contexte
BUT 1 — SAE

Contexte du projet

Dans le cadre de la SAE du semestre 2 du BUT Informatique, nous devions développer une version numérique du jeu de plateau japonais « Trains ». Ce projet constituait l'aboutissement de notre apprentissage en programmation orientée objet et représentait notre premier projet d'envergure impliquant une interface graphique complète.

Le projet était structuré en trois parties distinctes permettant d'évaluer différentes compétences : le développement du moteur de jeu, l'implémentation d'algorithmes sur les graphes, et la création de l'interface utilisateur.

Structure du projet

  • Partie 1 — Moteur de jeu : Développement de toute la logique du jeu en Java orienté objet. Gestion des règles, des cartes, des joueurs et du déroulement des parties.
  • Partie 2 — Algorithmes de graphe : Implémentation de fonctions pour calculer des éléments sur la carte du jeu (chemins, connexions, scores). Cette partie était davantage orientée vers l'évaluation des compétences algorithmiques.
  • Partie 3 — Interface graphique : Création d'une interface utilisateur complète en FXML permettant de jouer au jeu de manière intuitive et agréable.

Technologies utilisées

Java JavaFX FXML POO Algorithmes de graphe Git

Compétences développées

Réaliser

Développer une application complète en respectant les besoins du client

Optimiser

Implémenter des algorithmes efficaces sur les graphes

Collaborer

Travailler en binôme avec partage des tâches via Git

Concevoir

Modéliser une architecture orientée objet cohérente

Résultat

Le projet a abouti à un jeu fonctionnel permettant de jouer des parties complètes de Trains avec une interface graphique fluide. L'architecture orientée objet mise en place a permis d'avoir un code maintenable et extensible, facilitant l'ajout de nouvelles cartes ou règles.
