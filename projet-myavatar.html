MyAvatar

Clone du service Gravatar avec Symfony — Gestion d'avatars par email

Durée
~1 mois
Équipe
4 personnes
Framework
Symfony
🖼️

Contexte du projet

Ce projet de SAE au semestre 5 consistait à développer un clone du service Gravatar en utilisant le framework Symfony. Gravatar permet d'associer une photo de profil à une adresse email : n'importe quelle application peut ensuite charger cette image simplement en hachant l'email avec SHA256.

L'objectif était de maîtriser le développement web "server-side rendering" avec Symfony et Twig, tout en implémentant des fonctionnalités avancées comme l'authentification, la gestion de fichiers, et les commandes Symfony.

Fonctionnalités principales

  • Inscription/Connexion : Système complet d'authentification avec login, mot de passe et adresse email.
  • Gestion de profil : Upload, modification et suppression de photo de profil depuis un espace privé.
  • Accès public aux avatars : Route /avatar/{hash} permettant d'accéder à l'avatar via le hash SHA256 de l'email.
  • Profil masquable : Possibilité de masquer/démasquer son profil en AJAX sans rechargement de page.
  • Mode maintenance : Paramètre dans .env permettant de basculer le site en mode maintenance.
  • Administration : Rôle administrateur avec liste des utilisateurs et possibilité de suppression.
  • Commandes CLI : Commandes Symfony pour créer/supprimer des utilisateurs depuis le terminal.
  • Vérification AJAX : Validation en temps réel du login et email lors de l'inscription.

Points techniques notables

Un aspect intéressant du projet était la sécurisation du stockage des images. Les photos de profil ne devaient pas être accessibles publiquement dans le dossier public/, car un utilisateur peut masquer son profil. La solution a été de servir les images via un contrôleur Symfony qui vérifie les permissions avant de retourner le fichier.

Ce projet a également servi de base pour le projet MyCritique, qui utilise les avatars MyAvatar pour afficher les photos de profil des utilisateurs.
