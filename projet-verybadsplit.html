Very Bad Split

Clone de Tricount — Application de partage de dépenses entre amis

Durée
1 semestre
Équipe
5 personnes
Focus
Qualité & Architecture
💰

Contexte du projet

Cette SAE du semestre 4 avait une particularité : nous avons reçu un code existant volontairement mal conçu (d'où le nom "Very Bad Split") et notre mission était triple :

  • Analyser les failles de sécurité, bugs et problèmes de qualité du code existant
  • Normaliser la base de données en justifiant chaque décomposition
  • Améliorer l'application en corrigeant les problèmes et en ajoutant des fonctionnalités

L'application est un clone de Tricount permettant de créer des événements, inviter des membres, et gérer les dépenses partagées avec calcul automatique des dettes.

Fonctionnalités de l'application

  • Gestion des événements : Création, modification, suppression avec code secret de partage pour accès en lecture seule.
  • Gestion des membres : Invitation, éjection de membres avec gestion des droits propriétaire/membre.
  • Gestion des dépenses : CRUD complet avec répartition équitable entre participants.
  • Calcul des dettes : Algorithme optimisé minimisant le nombre de transactions nécessaires pour équilibrer les comptes.
  • Interface dynamique : Toutes les actions sur la page d'événement sans rechargement grâce à JavaScript + AJAX.
  • Réactivité : Mise à jour automatique de l'interface lors des modifications grâce à une bibliothèque réactive développée en cours.

Travail réalisé

Phase 1 — Analyse

Rédaction d'un rapport d'audit complet identifiant les failles de sécurité (injections SQL, XSS, CSRF...), les violations des principes de qualité logicielle (DRY, SOLID), et les anomalies de la base de données.

Phase 2 — Normalisation BDD

Passage du schéma initial dénormalisé à la 3ème forme normale avec justification de chaque décomposition, preuve de préservation des données et des dépendances fonctionnelles.

Phase 3 — Amélioration

Refonte complète de l'architecture avec routage, vues Twig, couche service, tests unitaires PHPUnit, API REST avec JWT, et dynamisation JavaScript.

Ma contribution

J'ai été fortement impliqué dans la partie back-end du projet :

  • Développement de l'API REST : Création des endpoints pour toutes les opérations (événements, membres, dépenses).
  • Authentification JWT : Mise en place du système d'authentification sécurisé.
  • Couche service : Implémentation de la logique métier dans des services testables.
  • Tests unitaires : Écriture de tests PHPUnit pour la couche métier.
  • Correction des failles : Sécurisation de l'application (requêtes préparées, échappement, validation).

Points techniques notables

Ce projet a été particulièrement formateur sur la qualité logicielle. Partir d'un code volontairement mal écrit nous a permis de comprendre concrètement l'importance des bonnes pratiques. L'implémentation des tests unitaires avec une couverture de 100% sur la couche métier a été un exercice exigeant mais très enrichissant.

La virtualisation avec Docker (un conteneur pour la BDD, un pour l'application) a également été une compétence clé acquise lors de ce projet.
